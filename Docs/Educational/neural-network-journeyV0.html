Foundations: What is Classification?

Teaching computers to recognize patterns

The Core Question

How do you teach a computer to guess what you're thinking?

Feature Vectors

Represent each animal as a list of numbers ‚Äî its "fingerprint"

Penguin: [0, 2, 0, 2] // fly, swim, mammal, bird Dog: [0, 0, 2, 0] Eagle: [2, 0, 0, 2] Shark: [0, 2, 0, 0]

Distance = Similarity

Similar animals have similar fingerprints.

Use Manhattan distance to measure how "far apart" two animals are.

üîë Key Insight

Identical fingerprints = indistinguishable objects. If Cat and Dog have the same feature vector, no algorithm can tell them apart. The quality of your questions determines the ceiling of any algorithm's intelligence.

MODULE 2 ‚úì

From Lookup to Learning

What if the computer figured out patterns itself?

The Neuron

Inspired by biology: a unit that takes inputs, weighs them, and fires (or doesn't).

output = sigmoid( Œ£(weight √ó input) + bias )

Weights = Knowledge

Weights encode what the neuron has learned about which features matter.

High weight = this feature is important for detecting this class.

The Learning Rule

weight += learningRate √ó error √ó input

If wrong, adjust weights in the direction that reduces error.

üîë Key Insight

Training is just adjusting weights until errors shrink. The neuron doesn't "understand" ‚Äî it just finds patterns in numbers through trial and error.

The Sigmoid Function

Squashing infinity into [0, 1]

sigmoid(x) = 1 / (1 + e-x)

Why Sigmoid?

Converts any number into a probability-like value between 0 and 1. Perfect for "is this a dog? yes/no" decisions.

Biological Inspiration

Real neurons either fire or don't. Sigmoid models this "soft" threshold ‚Äî values near 0.5 are uncertain, near 0 or 1 are confident.

MODULE 6 ‚úì

Backpropagation

How do errors flow backward through layers?

The Credit Assignment Problem

If the network guesses "Dog" when it should be "Cat"...

Which hidden neuron is to blame?

Output Layer: Easy

error = expected - actual // Direct supervision

Hidden Layer: Hard

error = ??? // No "expected output"!

The Solution: Blame Flows Backward

If hidden neuron H has a strong connection to output neuron O that has a big error...

Then H contributed heavily to that error!

hiddenError[h] = Œ£( outputError[o] √ó weight[h‚Üío] )

Backpropagation: The Flow

Forward Pass

Signals flow through weights ‚Üí activations ‚Üí outputs

Weights amplify signals

Backward Pass

Errors flow through same weights ‚Üí blame assignment

Weights distribute blame

üîë Key Insight

The same weights that carry signals forward also carry errors backward. It's a beautiful symmetry ‚Äî the network learns by reversing its own computation.

The Sigmoid Derivative

Why neurons near 0 or 1 learn slowly

derivative = output √ó (1 - output)

Output ‚âà 0.5

Derivative = 0.25 (maximum)

Neuron is uncertain ‚Äî very responsive to learning

Output ‚âà 0 or 1

Derivative ‚âà 0

Neuron is saturated ‚Äî confident, barely changes

üîë Key Insight

The derivative answers: "How much can this neuron actually change right now?" It prevents wasting updates on neurons that are already confident.

Backpropagation: The Code

public void Train(double[] inputs, double[] expected, double lr) { // STEP 1: Forward pass var hiddenOutputs = ComputeSingleLayer(inputs, Layers[0]); var finalOutputs = ComputeSingleLayer(hiddenOutputs, Layers[1]); // STEP 2: Output errors (with derivative!) var outputErrors = new double[outputLayer.Count]; for (int i = 0; i < outputLayer.Count; i++) { var o = finalOutputs[i]; outputErrors[i] = (expected[i] - o) * o * (1 - o); } // STEP 3: Hidden errors (backpropagation!) var hiddenErrors = new double[hiddenLayer.Count]; for (int h = 0; h < hiddenLayer.Count; h++) { double sum = 0; for (int o = 0; o < outputLayer.Count; o++) { sum += outputErrors[o] * outputLayer[o].Weights[h]; } var ho = hiddenOutputs[h]; hiddenErrors[h] = sum * ho * (1 - ho); } // STEP 4: Update weights for (int i = 0; i < outputLayer.Count; i++) outputLayer[i].Train(hiddenOutputs, lr, outputErrors[i]); for (int i = 0; i < hiddenLayer.Count; i++) hiddenLayer[i].Train(inputs, lr, hiddenErrors[i]); }
MODULE 7 ‚Äî UPCOMING

Training at Scale

How do you train on millions of examples efficiently?

Batch Training

Train on all examples, then update weights once. Stable but slow.

Stochastic (SGD)

Update after each example. Fast but noisy.

Mini-Batch

Best of both worlds. Industry standard.

Learning Rate Schedules

Start big, go small. Like cooling metal.

Concepts to Explore

  • Overfitting ‚Äî memorizing training data vs. learning patterns
  • Validation sets ‚Äî "trust but verify" with held-out data
  • Early stopping ‚Äî stop when validation error rises
MODULE 8 ‚Äî UPCOMING

Persistence & Production

Save and load a trained network

The Problem

Training takes millions of iterations. We don't want to start over every time!

What to Save

  • Network architecture (layer sizes)
  • All weights
  • All biases
  • Version/metadata

Options

  • JSON serialization (simple)
  • Database tables (structured)
  • Binary format (efficient)
üí° Key Concept

Training is expensive. Inference is cheap. Train once, deploy everywhere. This is why ML models are shipped as trained weights, not training code.

MODULE 9 ‚Äî OPTIONAL

Smart Question Selection

What question gives the most information?

The Idea

Not all questions are equal. "Does it fly?" might be useless if you've already narrowed to aquatic animals.

Information Gain = Entropy(before) - Entropy(after)

Entropy

Measure of uncertainty. High entropy = many possibilities. Low entropy = almost certain.

Information Gain

How much a question reduces uncertainty. Pick the question that maximizes this!

üí° Key Concept

Decision trees vs neural networks ‚Äî Decision trees explicitly optimize question order. Neural networks learn features implicitly. Both have tradeoffs.

Key Lessons Learned

Bugs that taught us the most

Reference vs Value Types

All neurons shared the same weights array because arrays are reference types. Fix: create new array inside the loop.

Loop Variable Scope

`var i = 0` inside foreach resets every iteration. Fix: declare outside loop.

Random Initialization

`new Random()` in tight loops produces same values. Fix: create Random once, reuse.

Identical Feature Vectors

Cat and Dog were indistinguishable. Fix: add discriminating questions.

Missing Sigmoid Derivative

Network didn't generalize well. Fix: scale errors by derivative.
